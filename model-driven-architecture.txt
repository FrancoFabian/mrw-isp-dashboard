Propuesta de Arquitectura Model‑Driven (MDA)

Para replicar o mejorar estas funcionalidades, una arquitectura MDA puede estructurarse en módulos conectados por entidades y procesos:

Módulo	Entidades/Modelos clave	Funcionalidades sugeridas
Clientes y contratos	Cliente, Contrato, Plan de servicio, Dirección, Nodo/NAP	CRUD de clientes y contratos; asignación de planes y equipos; estados (activo, suspendido, deshabilitado); historial de cambios.
Equipos y red	OLT, ONU, NAP, Puerto, Equipo CPE, Firmware	Inventario de OLT/ONU/CPE; gestión de puertos y ONUs; potencias de señal; asignación de equipos; actualización remota (TR‑069); geolocalización.
Instalaciones y órdenes	Orden de instalación, Técnico, Agenda, Precinto FTTH	Gestión de órdenes, calendarización, asignación de técnicos, registro de material utilizado, generación de contratos PDF, factibilidad.
Facturación y pagos	Factura, Ciclo de facturación, Pasarela de pago, Medio de pago, Pago, Promesa de pago, Impuesto/Descuento	Emisión de facturas electrónicas (adaptadas por país); configuración de ciclos; integración con múltiples pasarelas; control de impuestos/descuentos; envío de facturas (WhatsApp/email/SMS); conciliación de pagos; promesas de pago; estados (emitida/pagada/vencida).
Aplicaciones móviles	Sesión móvil, Notificación, Ticket (soporte), Promesa de pago, Comprobante de pago	Ver y pagar facturas; crear promesas y tickets; cambio de contraseña Wi‑Fi; notificaciones push; gestión multi‑cuenta con QR; gráficas de consumo; canales de contacto.
Soporte y tickets	Ticket, Categoría, Estado, Mensajes, Archivo adjunto	Registro de problemas, seguimiento y resoluciones; asignación de responsables; comunicación con el cliente; estados (pendiente/resuelto).
Comunicaciones omnicanal	Conversación, Canal (WhatsApp, Instagram, Facebook, Teléfono), Plantilla de mensaje, Etiqueta, Agente	Centralización de chats; automatización de mensajes; registro histórico; asignación a áreas/equipos; métricas (tiempo de respuesta, SLA); integraciones con API oficial de WhatsApp.
Notificaciones y recordatorios	Evento, Tipo de notificación, Medio (push, email, WhatsApp), Programación	Notificaciones automáticas de facturas emitidas/vencimientos/pagos/tickets; mensajes personalizados; campañas de marketing; configuración de reglas por cliente/plan.
Integraciones externas	Servicio de facturación electrónica, Pasarelas de pago, API TR‑069, API WhatsApp	Abstracción de servicios externos; control de versiones; monitoreo de integraciones; fallback y errores.
